# TOOLS - 可用工具列表

## get_system_info
- **描述**: 获取当前容器的操作系统版本与 Python 版本
- **参数**: 无需参数
- **返回**: 操作系统名称/版本、架构、Python 版本

## get_current_time
- **描述**: 获取当前本地时间
- **参数**: 无需参数
- **返回**: YYYY-MM-DD HH:MM:SS 格式的时间字符串

## read
- **描述**: 读取指定路径的文本文件内容
- **参数**: 文件路径（绝对路径或相对路径）
- **返回**: 文件内容（文件过长时自动截断至 5000 字符）

## write_file
- **描述**: 写入或覆盖一个文本文件
- **参数**: `文件路径|||文件内容`（用三个竖线 `|||` 作为分隔符）
- **示例**: `/app/workspace/note.md|||这是新内容`
- **返回**: 写入成功提示 + 文件大小，或错误信息
- **注意**: 父目录不存在时会自动创建；已存在文件会被覆盖

## edit
- **描述**: 对已存在文件做局部修改（查找 old_text 替换为 new_text），无需重写整个文件
- **参数**: `文件路径|||old_text|||new_text`（用三个竖线 `|||` 分隔 3 段）
- **示例**:
  - 替换：`/app/main.py|||import os|||import sys`
  - 删除：`/tmp/a.txt|||要删除的片段|||`（new_text 留空）
- **返回**: 操作类型 + 字符变化统计，或错误信息
- **注意**: old_text 必须在文件中**恰好出现一次**，否则报错（防止误改）；若多处同名，需扩大上下文使其唯一

## bash
- **描述**: 在容器中执行一条 bash 命令
- **参数**: 完整的命令字符串，例如 `ls -la /app/workspace` 或 `pip list | grep openai`
- **返回**: `[exit code] + [stdout] + [stderr]` 组合文本
- **限制**: 单条命令超时 30 秒；输出超 4000 字符会被截断
- **注意**: 每次调用都是独立子进程，不保留工作目录或环境变量变更（`cd` 的效果不会延续）

## todo
- **描述**: 管理多步任务的 todo 列表（带状态：pending / in_progress / completed / cancelled）
- **参数**: JSON 数组字符串；每项含 `id`、`text`、`status`。无参时返回当前列表。
- **示例**: `[{"id":"1","text":"读取 SOUL.md","status":"completed"},{"id":"2","text":"写入诗","status":"in_progress"}]`
- **约束**: 同一时间最多 1 个 `in_progress`
- **建议**: 2 步以上的任务**先规划再执行**；每步完成后立即更新状态

## 使用约束
- 所有路径参数建议使用绝对路径以避免歧义
- read 工具无法读取目录或二进制文件
- 工具执行失败时返回错误信息而非抛出异常
- bash 工具请谨慎使用破坏性命令（rm、mv 等）
